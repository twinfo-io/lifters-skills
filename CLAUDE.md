# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Purpose

`lifters-skills` is the central repository of Claude Code skills for Lifters teams. It serves two purposes:

1. **Skills distribution** вЂ” Claude Code commands (`/discovery`, `/new-feature`) installed company-wide via `npx skills add twinfo-io/lifters-skills`, available in every project on the developer's machine.

2. **Specification workspace** вЂ” Houses the `ai/specs/` directory where feature specs for this repository itself are maintained.

## Format

This repository follows the **Agent Skills** standard (`agentskills.io`). Each skill lives in its own directory under `skills/` with a `SKILL.md` file containing YAML frontmatter and the skill instructions.

## Installation

```bash
npx skills add twinfo-io/lifters-skills       # via registry (recommended)
curl -sSL https://raw.githubusercontent.com/twinfo-io/lifters-skills/main/install.sh | bash  # fallback
```

Skills are installed to `~/.claude/skills/` (user-level) вЂ” not project-level.

## Commands

### `/discovery [feature description]`

Starts an interactive feature discovery. Collects input documents first (URLs, pasted content, multiple docs), then creates the folder structure, conducts a greenfield/brownfield interview, researches market references automatically, and generates `discovery.md`.

```
/discovery "CobranГ§a recorrente com Stripe"
/discovery   # without argument вЂ” asks for description first
```

Output: `ai/specs/YYYYMMDDHHmmSS_nome/discovery.md`

### `/new-feature`

Reads an existing `discovery.md` (or conducts inline discovery if none exists) and generates the three canonical artifacts.

Output:
- `ai/specs/YYYYMMDDHHmmSS_nome/briefings/briefing.v0.md`
- `ai/specs/YYYYMMDDHHmmSS_nome/specs.md`
- `ai/specs/YYYYMMDDHHmmSS_nome/wps.md`

## Adding New Skills

1. Create `skills/skill-name/SKILL.md` with YAML frontmatter (`name` + `description`) and the skill instructions
2. Add templates to `skills/skill-name/templates/` if needed
3. Add the entry to `skills.json`
4. Document in `README.md` and `CHANGELOG.md`
5. Open a PR for team review

**Required frontmatter:**
```yaml
---
name: skill-name
description: What it does and when to use it (max 1024 chars, no XML tags)
---
```

Naming convention: `kebab-case`, imperative English verbs (`gen-docs`, `review-pr`).
